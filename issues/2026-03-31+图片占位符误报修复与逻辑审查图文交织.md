# 图片占位符误报修复 & 逻辑审查图文交织

## 背景

Mammoth 将 DOCX 内的图片转换为 `![图N]()` 占位符（N 为文档顺序编号），并将其插入 Markdown
中图片在 DOCX XML 锚定位置处。当图片以浮动方式锚定在文字段落内时，占位符会插入句子中间，
导致两类问题：

1. **格式审查误报**：LLM 收到 `Encoder-Decoder ![图2]()Attention`，误判"图片切断专业术语"；
   占位符序号 `图2` ≠ 论文图题 `图2-2`，误判"图表与正文不一致"（`figure_table_mismatch`）。
2. **逻辑审查图文脱节**：所有图片堆在消息末尾，LLM 无法将文字内容与对应图片关联阅读。

## 修复方案

### 修复一：格式审查 strip 占位符

**文件**：`lib/review/hybrid-docx-parser.ts`、`lib/services/review/format.service.ts`

- 新增 `stripDocxImagePlaceholders(markdown)` 函数，正则 `!\[[^\]]*\]\(\)` 剥离所有 Mammoth 图片占位符。
- 在 `analyzeFormat` 内，调用 `extractGlobalSkeleton` / `splitMarkdownByChapters` 前先 strip，
  得到 `cleanMarkdown`，去除 LLM 不可见的图片噪声。
- 不修改解析层（`parseHybridDocx` 返回的 `markdown` 不变），逻辑审查仍可拿到原始带占位符的 markdown。

### 修复二：逻辑审查图文交织

**文件**：`lib/services/review/review-messages.ts`、`lib/services/review/logic.service.ts`

- 新增 `buildDocxInterleavedMessages(markdownText, images)`：
  - 按 `![图(\d+)]()` 切割 markdown
  - 文字段 → `{ type: "text" }` part
  - 占位符 → 查 `images`（按 `order`）→ `{ type: "image" }` part，图文交织
  - 未命中的图（超限/失败）自然跳过，不产生空 part
- `logic.service.ts` 将 `buildDocxMultimodalMessages` 替换为 `buildDocxInterleavedMessages`。

## 改动文件清单

| 文件 | 改动类型 |
|------|---------|
| `lib/review/hybrid-docx-parser.ts` | 新增导出函数 `stripDocxImagePlaceholders` |
| `lib/services/review/format.service.ts` | 导入并在 `analyzeFormat` 内应用 strip |
| `lib/services/review/review-messages.ts` | 新增 `buildDocxInterleavedMessages` |
| `lib/services/review/logic.service.ts` | 切换到 `buildDocxInterleavedMessages` |
