# 逻辑审查 Prompt 优化：提升问题发现量

## 背景
当前 logic_review_pass1/pass2 prompt 返回的问题数量偏少（通常 5-10 条），需要在不增加调用轮次、不改变 JSON 返回结构的前提下，通过优化 prompt 提升问题发现量。

## 改进策略
1. **数量锚定**：利用 LLM 锚定效应，设置 20-30 条的数量预期
2. **扩展 issue_type 子维度**：为 5 个类型各增加具体审查角度
3. **输出前自检指令**：要求模型检查覆盖完整性
4. **temperature 微调**：pass1 0.38→0.45，pass2 0.42→0.48
5. **pass2 补漏强化**：增加更多具体补漏角度和数量预期

## 涉及文件
- `config/prompts.default.json`：logic_review_pass1 (v1.2.0→v1.3.0)、logic_review_pass2 (v1.1.0→v1.2.0)

## 预期效果
- pass1：5-10 条 → 15-25 条
- pass2：2-5 条 → 5-15 条
- 合计：20-35+ 条
