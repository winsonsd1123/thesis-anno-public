# 学术数据库标题搜索 limit=1 导致精确文献漏判修复

## 问题
CrossRef / OpenAlex / Semantic Scholar 三个数据库的标题搜索均使用 `limit=1`（即只取相关性排名第 1 的结果），若目标文献排在第 2 位及以后，精确标题匹配失败 → `database_candidate=null` → LLM 误判为"存疑/未找到"。

典型案例：
- #22 "Histograms of oriented gradients for human detection"（Dalal & Triggs, CVPR'05）在 CrossRef 搜索结果中排名第 2，被第 1 名不相关论文挤掉。
- #19 "Cat face recognition using deep learning"（ASABE 会议，小众会议 CrossRef/OpenAlex 未收录，Semantic Scholar 有但未命中）。

## 修复

| 文件 | 变更 |
|------|------|
| `crossref.ts` | `rows=1` → `rows=5`，遍历 items 找精确匹配 |
| `openalex.ts` | `per_page=1` → `per_page=5`，遍历 results 找精确匹配 |
| `semantic-scholar.ts` | `limit=1` → `limit=5`，遍历 data 找精确匹配 |

匹配逻辑均移至各搜索函数内部（`titlesExactMatchNormalized`），找到第一个精确匹配即返回，未命中全部 5 条则返回 null。
waterfall.ts 的 `firstTitleSearchExactTitle` 包装层保持不变（冗余守护）。
