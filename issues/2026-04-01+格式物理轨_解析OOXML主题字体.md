# 格式物理轨：解析 OOXML 主题字体（eastAsiaTheme）

## 背景

格式检查引擎在 run 级检查时报：
> 规则 body-default（run 级）：期望中文字体在 [宋体]，实际为 (无)

根因：OOXML 解析器仅读取 `w:rFonts` 的 `eastAsia` 显式字体属性，
不读 `eastAsiaTheme`（如 `minorEastAsia`）。现代 Word 文档默认使用主题字体，
XML 中只有 `eastAsiaTheme` 而无 `eastAsia`，导致 `font_zh` 解析为 `undefined`。

额外发现 1：中文版 Office 的 theme1.xml 中 `<a:ea typeface="">` 通常为空字符串，
实际中文字体放在 `<a:font script="Hans" typeface="等线"/>` 等 script-specific 节点中。

额外发现 2：当段落没有显式 `w:pStyle` 时，解析器直接 fallback 到 `docDefaultsRPr`，
跳过了 `w:default="1"` 的默认段落样式（通常为 Normal）。Normal 样式中的 `eastAsia="宋体"`
被忽略，导致 run 的字体被错误解析为主题默认字体"等线"。

## 方案

从 ZIP 中读取 `word/theme/theme1.xml`，解析主题字体映射（major/minor × ea/latin），
在 `extractRPr` 中读取 theme 引用属性，最终在 `rprToRunSpan` / `paragraphToNode`
中用 `resolveThemeFont` 回填实际字体名。显式字体优先于主题字体（符合 OOXML 规范）。
当 `<a:ea>` 为空时，回退到 `<a:font script="Hans">` 取简体中文字体。
同时修复：当段落无 `pStyle` 时，使用 `w:default="1"` 的默认段落样式解析字体链。
