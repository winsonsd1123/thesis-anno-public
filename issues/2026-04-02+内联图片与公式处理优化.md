# 内联图片占位符与数学公式处理优化

## 问题

1. **图片占位符误报**：`stripDocxImagePlaceholders` 将 `![图N]()` 完全删除后送给 LLM，LLM 看不到图片存在，误报 `figure_table_mismatch`（"缺少图形可视化内容"）。
2. **公式段落误检**：纯数学公式段落在 AST 中 `text` 为空，但 `attachMathToStyleAst` 后未回填文本；若后续填充文本，物理规则引擎会对公式段落进行字体/字号检查（公式用 Cambria Math 等专用字体，必然报错）。

## 方案

### A1：图片占位符 → 纯文本标记

- `stripDocxImagePlaceholders` 重命名为 `neutralizeDocxImagePlaceholders`
- 替换逻辑：`![alt]()` → `[此处有嵌入图片]`（不保留 Mammoth 的顺序编号，避免与论文实际图号冲突）
- Prompt 新增第 7 条"嵌入图片说明"，告知 LLM 该标记表示实际图片存在

### B1：公式段落文本填充 + 物理规则跳过

- `parseHybridDocx` 中 `attachMathToStyleAst` 后，遍历 styleAst：纯公式段落（`text` 为空且 `math_latex` 非空）填充 `text = "[数学公式] " + latex`
- `format-rules.engine.ts` 主循环增加 `math_latex` 判断，跳过公式段落

## 改动文件

| 文件 | 改动 |
|------|------|
| `lib/review/hybrid-docx-parser.ts` | 重命名函数 + 替换逻辑 + 公式段落文本填充 |
| `lib/services/review/format.service.ts` | 更新导入 |
| `config/prompts.default.json` | local prompt 新增第 7 条嵌入图片说明 |
| `lib/services/review/format-rules.engine.ts` | 跳过 `math_latex` 段落 |

## 验证

- 17 个 AST 测试 + 24 个 Math 测试全部通过
- 马锐文论文端到端验证：16 张图片全部替换为标记，16 个纯公式段落已填充文本，物理规则零误检

## 补充：内联公式位置（2026-04-02）

**问题**：同一段落内 `w:r` 与 `m:oMath` 交替时，原逻辑把 LaTeX 追加到 Mammoth 匹配行**末尾**，且 `findLineIndex` 的 `norm.includes(短行)` 会把**长段落**误匹配到先行短行，导致公式跑到错误位置（如「图3.5」单独一行）。

**改动**（`docx-math-extractor.ts`）：

1. `MathFragment.textWithMath`：按段落子节点顺序拼接「文字 + `$...$`」。
2. `appendMathToMarkdown`：有 `textWithMath` 时用**最长后缀**在匹配行内替换（兼容段首被 Mammoth 收成 `![图N]()` 的情况）。
3. `findLineIndexForInlineReplace`：仅选用「含足够长 `paragraphText` 后缀」的行，取**最长后缀**对应行，避免短行误匹配。
4. `textWithMath` 存在但替换失败时进附录，**不再**在误匹配行尾追加公式。
